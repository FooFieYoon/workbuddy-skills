# 通用学术论文全流程写作助手

## 工作流总览

```
[Phase 0] 收集输入（投稿要求/选题/学科/目标期刊/字数限制）
        ↓
[Phase 1] 选题分析 → 文献检索 → 选题推荐（3-5个）→ 用户确认
        ↓
[Phase 2] 深度文献检索（政策/期刊论文/案例/数据/理论）
        ↓
[Phase 3] 生成大纲 → 用户确认 → 逐章生成正文
        ↓
[Phase 4] 图表识别与生成
        ↓
[Phase 5] Word 文档排版输出
        ↓
[Phase 6] 用户反馈 → 迭代修改 → 最终成稿
```

---

## Phase 0：收集写作输入

### 0.1 从用户输入中提取以下信息

| 信息项 | 说明 | 示例 |
|--------|------|------|
| 学科领域 | 论文所属学科 | 高等教育、计算机科学、管理学、医学 |
| 目标期刊/会议 | 投稿目标 | 《中国高教研究》、IEEE CVPR、ACM CHI |
| 投稿要求 | 字数、结构、格式要求 | 6000-10000字、IEEE 双栏格式 |
| 稿件类型 | 论文类型 | 研究论文、综述、案例研究、短文 |
| 研究背景 | 作者研究方向/所在机构/已有资源 | 某大学教授，从事 NLP 研究5年 |
| 截稿日期 | 提交截止（如有） | 2026年X月X日 |

若用户未提供以上信息，主动询问学科领域和目标期刊/会议（最重要的两项）。

### 0.2 风格规范加载

- **用户提供了目标期刊/会议的示例论文或模板**：使用 Python 或 `pdfkit-py` skill 提取文本，分析风格规范
- **用户未提供示例**：加载 `references/default-style-guide.md` 中的通用学术论文规范

## Phase 1：选题推荐

### 1.1 文献检索策略

围绕学科领域和投稿要求，执行至少 **15-20 次 WebSearch**，覆盖：

```
# 政策/行业背景（适用于社科/工程/教育等应用学科）
搜索词："{研究领域核心词}" 政策 2023 2024 2025 site:gov.cn

# 顶级期刊/会议近3年论文
搜索词："{关键词}" "{目标期刊名}" 2023 2024 2025
搜索词："{关键词}" site:cnki.net 核心期刊 2024 2025
搜索词："{English keyword}" "{Target Venue}" 2023 2024 2025

# 综述与热点趋势
搜索词："{研究领域}" 综述 现状 挑战 进展 2024 2025
搜索词："{research topic}" survey review state-of-the-art 2024 2025

# 案例/实践（应用研究）
搜索词："{研究对象}" 案例 实践 创新 经验 2024 2025
```

### 1.2 选题推荐格式

每个选题提供：

```markdown
### 选题N：{论文标题}

**副标题**（可选）：——{基于XX的实证研究 / 以XX为例 / XX视角下的XX}

**论文类型**：
- 实证研究型 / 综述型 / 案例研究型 / 理论构建型 / 方法提出型

**核心贡献**：
- 本文的主要研究问题是什么
- 提出了什么新发现/模型/方法/框架
- 对领域的理论或实践价值

**可行支撑**：
- 已找到 X 篇相关核心文献
- 已找到 X 条政策/行业数据支撑
- 作者现有资源/数据可以支持此选题

**初步参考文献**：
1. 作者. 标题[J/C]. 期刊/会议, 年份: 页码.
```

### 1.3 用户确认

使用 `AskUserQuestion` 让用户选择或调整选题，确认后进入 Phase 2。

## Phase 2：深度文献检索

选题确认后，围绕该选题系统化检索，**至少执行 20 次 WebSearch**：

| 类别 | 目标 | 工具 |
|------|------|------|
| 理论基础 | 相关理论/模型的经典文献 | WebSearch + WebFetch |
| 核心期刊论文 | 每个子方向至少5篇近3年论文 | WebSearch + WebFetch |
| 数据/统计 | 统计公报、行业报告、政府数据 | WebSearch + WebFetch |
| 案例/实验 | 典型案例、已发表实验结果 | WebSearch |
| 批评/争议 | 现有研究的局限性文献 | WebSearch |
| 国际比较（如需） | 国外同类研究 | WebSearch（英文） |

整理为结构化资料清单展示给用户，然后继续。

## Phase 3：生成大纲与正文

### 3.1 大纲结构选择

根据论文类型，选择对应大纲结构（详见 `references/paper-structures.md`）：
- **实证研究**：引言→文献综述→研究假设/模型→数据与方法→结果→讨论→结论
- **方法提出**：引言→相关工作→方法→实验设置→实验结果→分析→结论
- **综述论文**：引言→检索策略→现有研究分类综述→研究现状与挑战→未来方向→结论
- **案例研究**：引言→理论框架→案例背景→案例分析→发现→讨论→结论
- **理论构建**：引言→概念界定→理论框架构建→命题推导→讨论→结论

展示大纲给用户确认后再生成正文。

### 3.2 正文生成规范

逐章节生成，每节遵守：

**引言章节（三段标准结构）**：
1. 研究背景：领域重要性，引用近期政策/报告/数字
2. 研究问题：现有研究空白/局限性（Research Gap）
3. 论文贡献：本文的方法、结论、意义，结构说明

**文献综述/相关工作**：
- 按主题分类综述，而非按时间列举
- 每段末尾明确说明本文与已有研究的区别
- 引用格式与目标期刊保持一致

**核心章节（方法/实践/机制）**：
- 章节内容占全文 35-45%
- 逻辑：问题→方案→机制/算法→实现细节
- 必须包含: 具体参数/名称/数据/步骤

**结果/成效章节**：
- 量化指标（数字、百分比、对比数据）
- 与引言研究问题形成闭环呼应

**讨论章节**：
- 解释为何得到这些结果
- 比较已有研究，说明贡献
- 诚实陈述局限性

**结论章节**：
- 简要总结各章主要发现（2-3句/章）
- 升华到领域或实践价值
- 未来工作方向（2-3条）

### 3.3 语言与写作风格

使用 `references/writing-style.md` 中的风格指南（包含各学科领域的差异规范）。

**通用原则**：
- 避免第一人称（除非目标期刊明确允许）
- 每段有明确的主题句
- 论断必须有引用支撑
- 数字数据必须有来源

### 3.4 参考文献管理

- 按正文引用顺序编号（[1][2]...）或按作者年份（Author, year）
- 格式依目标期刊规范（GB/T 7714、APA、IEEE、Vancouver 等），详见 `references/citation-formats.md`
- 近5年文献占比不少于50%
- 正文引用位置用上标或括号标注

## Phase 4：图表匹配与生成

### 4.1 图表类型识别

| 内容类型 | 推荐图表 | 生成工具 |
|---------|---------|---------|
| 流程/机制描述 | 流程图/框架图 | `read_me(diagram)` + `show_widget` |
| 数值对比 | 柱状图/折线图 | `read_me(chart)` + `show_widget` |
| 系统架构 | 架构图 | `read_me(diagram)` + `show_widget` |
| 时间序列 | 折线图 | `read_me(chart)` + `show_widget` |
| 多维评价 | 雷达图 | `read_me(chart)` + `show_widget` |
| 概念关系 | 概念图/思维导图 | `read_me(diagram)` + `show_widget` |
| 对比分析 | 表格 | 直接 Markdown 表格 |

### 4.2 图表规范

- **图题**：`图X  {图题名称}`，位于图下方，居中
- **表题**：`表X  {表题名称}`，位于表上方，居中
- **编号**：图1、图2...，表1、表2...，全文连续
- **先文后图**：正文先写"如图X所示"再放图表

## Phase 5：Word 文档排版输出

### 5.1 排版优先级

1. **用户提供了模板文件**（.docx）：读取模板，提取字体/字号/间距/页面设置，严格按模板排版
2. **目标期刊提供了格式要求**：按格式要求设置
3. **无特殊要求**：使用 `references/default-style-guide.md` 中的通用排版规范

### 5.2 文档生成工具选择

- 优先调用 `docx` skill 生成 Word 文档
- 若需要精细排版，调用 `minimax-docx` skill

### 5.3 文档内容顺序

标题 → 作者及单位 → 摘要 → 关键词 → 正文（各章节） → 参考文献 → （附录）

## Phase 6：迭代修改

### 6.1 反馈处理

| 反馈类型 | 处理方式 |
|---------|---------|
| 内容修改/补充 | 重新搜索资料，修改对应章节 |
| 结构调整 | 重组章节，更新大纲 |
| 数据更新 | WebSearch 查找最新数据 |
| 语言润色 | 按目标语言风格修改 |
| 字数控制 | 扩写或精简指定章节 |
| 参考文献补充 | 补充搜索相关文献 |
| 降低 AIGC 检测率 | 调用 `humanize-ai-text` 或 `removeai` skill |

### 6.2 交付前质量自检

- [ ] 标题简洁准确，体现核心贡献
- [ ] 摘要独立完整，包含背景/问题/方法/结论/价值
- [ ] 关键词 3-8 个，符合领域术语
- [ ] 引言清晰说明研究背景、问题、贡献
- [ ] 核心章节（方法/机制）占全文 35-45%
- [ ] 结论与引言研究问题形成呼应
- [ ] 所有论断有文献支撑
- [ ] 量化数据有来源
- [ ] 参考文献格式统一，近5年文献 ≥50%
- [ ] 图表编号连续，图题表题位置正确
- [ ] 全文字数符合投稿要求

## 注意事项

1. **用户确认不可跳过**：选题和大纲必须经用户确认后才进入深度写作
2. **文献引用必须真实**：通过 WebFetch 核实文献标题、作者、年份，不伪造引用
3. **数据必须标注来源**：所有数字数据写明出处
4. **领域适配**：不同学科（自然科学、社科、工程、医学）的写作规范差异较大，参考 `references/paper-structures.md` 和 `references/writing-style.md` 中的学科差异说明
5. **字数控制**：根据投稿要求调整各章节篇幅分配
6. **搜索覆盖**：选题阶段 ≥15 次搜索，深度检索阶段 ≥20 次搜索
